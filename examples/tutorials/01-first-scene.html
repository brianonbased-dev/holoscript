Your First VR Scene in 5 Minutes

Learn HoloScript by building a glowing, spinning cube in virtual reality.
TUTORIAL 01 -- BEGINNER

HoloScript is a declarative language for building immersive 3D and VR experiences. Instead of wrestling with boilerplate, you describe what you want -- objects, environments, and behaviors -- and HoloScript takes care of the rest. In this tutorial you will create a complete scene from scratch.

Create Your .holo File

Every HoloScript scene lives in a .holo file. Create a new file called my-first-scene.holo and start with a composition block. The composition is the root container for your entire scene.

my-first-scene.holo Step 1 of 5 
// My very first HoloScript scene
composition "My First Scene" {

}
composition declares a named scene. Everything -- objects, environments, logic -- lives inside its curly braces. Think of it as the stage for your VR world.

Set Up the Environment

The environment block controls the world around your objects: the sky, lighting, and atmosphere. Let's add a gradient skybox and soft ambient light.

my-first-scene.holo Step 2 of 5 
composition "My First Scene" {
  environment {
    skybox: "gradient"
    ambient_light: 0.4
  }
}
skybox sets the background. Options include "gradient", "stars", "office", and custom HDRI paths.
ambient_light sets the base illumination level from 0 (pitch black) to 1 (fully lit).

Add Your First Object

Objects live inside spatial groups, which organize your scene into logical clusters. Let's create a group called "Main" and place a cyan cube inside it with two traits: @spinning and @glowing.

my-first-scene.holo Step 3 of 5 
composition "My First Scene" {
  environment {
    skybox: "gradient"
    ambient_light: 0.4
  }

  spatial_group "Main" {
    object "MyCube" @spinning @glowing {
      geometry: "cube"
      color: "#00ffff"
      position: [0, 1.5, -3]
      scale: [0.5, 0.5, 0.5]
    }
  }
}
spatial_group organizes related objects. You can nest groups, move them together, and toggle visibility per group.

object defines a 3D entity. The name ("MyCube") is how you reference it elsewhere in your code.

@spinning and @glowing are traits -- reusable behaviors you attach with the @ prefix. Traits are the heart of HoloScript; they let you compose complex behavior without writing low-level code.
Tip

Traits can be placed on the same line as the object declaration (object "Name" @trait1 @trait2 { ... }) or on separate lines inside the block. Both styles are valid.

Understanding Properties

Each object has properties that control how it looks and where it sits in the world. Here is what each property in our cube means:

geometry -- the shape: "cube", "sphere", "cylinder", "plane", or a path to a .glb model.

color -- a hex color string. "#00ffff" is cyan.

position -- an [x, y, z] array in meters. [0, 1.5, -3] places the cube at eye level, 3 meters in front of the viewer.

scale -- the size multiplier. [0.5, 0.5, 0.5] makes a half-meter cube. You can also use a single number for uniform scale.

Enhance with More Traits

The power of HoloScript is in its trait system. Let's add more interactivity by expanding our scene with additional objects and traits.

my-first-scene.holo -- final version Step 5 of 5 
// My First Scene - Complete Version
composition "My First Scene" {
  environment {
    skybox: "gradient"
    ambient_light: 0.4
  }

  spatial_group "Main" {
    // A glowing, spinning cube
    object "MyCube" @spinning @glowing {
      geometry: "cube"
      color: "#00ffff"
      position: [0, 1.5, -3]
      scale: [0.5, 0.5, 0.5]
    }

    // A clickable sphere that reacts to interaction
    object "MySphere" @clickable @physics @glowing {
      geometry: "sphere"
      color: "#ff6b6b"
      position: [2, 1.5, -3]
      scale: 0.4
    }

    // A transparent, floating cylinder
    object "MyRing" @floating @transparent {
      geometry: "cylinder"
      color: "#ffe66d"
      position: [-2, 1.5, -3]
      scale: [0.6, 0.1, 0.6]
      opacity: 0.6
    }
  }

  // Ground plane so objects have something to rest on
  object "Floor" @collidable {
    geometry: "plane"
    position: [0, 0, 0]
    scale: [10, 1, 10]
    color: "#1a1a2e"
  }
}

Common Traits Reference

Traits are the building blocks of interactivity. Attach them to any object with the @ prefix. Here are the most commonly used ones:

@grabbable
Users can pick up and move this object with their hands or controllers.
@glowing
Adds an emissive glow effect using the object's color.
@physics
Enables rigid-body physics: gravity, mass, collisions.
@clickable
Responds to click or tap events. Pair with on_click handlers.
@spinning
Continuously rotates the object around its Y axis.
@floating
Gently bobs the object up and down with a sine wave motion.
@transparent
Enables alpha transparency. Control with the opacity property.
@collidable
Gives the object a collision boundary. Other physics objects will bounce off it.
@animated
Plays built-in or custom keyframe animations on the object.
@networked
Synchronizes this object's state across all connected clients.

Running Your Scene

To load and execute your .holo file in the browser, include the HoloScript runtime bundle and call createRuntime():

index.html Runtime Setup 
<!-- Include the HoloScript IIFE bundle -->
<script src="holoscript.global.global.js"></script>

<script>
  // Create a runtime instance
  const runtime = window.HoloScript.createRuntime();

  // Load and run your scene
  fetch('my-first-scene.holo')
    .then(r => r.text())
    .then(source => {
      runtime.parse(source);
      runtime.execute();
    });
</script>
Tip

The runtime is available via the global window.HoloScript namespace after loading holoscript.global.global.js. No module bundler required for simple projects.
